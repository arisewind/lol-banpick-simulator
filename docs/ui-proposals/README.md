# UI 全面调整方案集

4 份风格迥异的 UI 调整方案，供选型实施。所有方案**保持现有功能不变**（三面板 / 导入导出 / 数据分析 / 语言切换 / error 重试），仅调整视觉与布局。每份方案均含：设计理念、配色系统（hex + CSS 变量 + 与现有 lol-* 映射）、字体、布局结构（ASCII 示意）、关键组件视觉（真实 Tailwind 类名）、交互动效、Tailwind 实现要点、迁移成本评估。

## 方案总览

| 方案 | 风格定位 | 配色核心 | 字体 | 迁移成本 | 适合场景 |
|------|---------|---------|------|---------|---------|
| [客户端仪式复刻](client-ceremony.md) | 赛事直播仪式感 | 金 `#c8aa6e` + 冷青/暗红（降饱和） | Cinzel + Chakra Petch | 中 | 贴近 LoL 客户端、强调 BP 庄重登场感 |
| [赛博朋克霓虹](cyberpunk-neon.md) | 未来战术终端 | 霓虹青/品红/紫 + 纯黑 `#0A0A0F` | Orbitron + JetBrains Mono | 中 | 强烈视觉冲击、赛博朋克美学、记忆点深 |
| [玻璃拟态深空](glass-deep-space.md) | 精致悬浮玻璃 | 深空渐变 + 内发光 + `bg-white/5` | Space Grotesk + Inter | 中偏低 | 现代精致、柔和层次、复用现有资产最多 |
| [极简数据流](minimal-data.md) | 战术数据终端 | zinc 灰阶 + 单琥珀 `#f59e0b` 强调 | Inter + JetBrains Mono | 中 | 硬核玩家、数据导向、分析面板升为主角 |

## 共同特点

- **保留蓝/红/金语义**：全部保留或映射 `lol-blue/red/gold` 令牌，队伍色语义不丢失
- **守 CLAUDE.md Tailwind 坑**：字面量类名（禁 `${}` 插值）、自定义 boxShadow 档位（禁 `/N`）、颜色阴影配尺寸类、`@import` 顺序、改 `tailwind.config.js` 需重启 Vite
- **尊重 `prefers-reduced-motion`**：动效均有降级
- **零逻辑改动**：纯视觉/布局调整，现有 79 个测试不受影响

## 选型建议

- **最低成本、最快见效** → [玻璃拟态深空](glass-deep-space.md)：中偏低，复用现有 glow/shadow 资产最多，可分 4 阶段渐进
- **差异化最强、记忆点最深** → [赛博朋克霓虹](cyberpunk-neon.md)：视觉冲击最强，纯黑 + 霓虹三色 + CRT 扫描线
- **贴近真实游戏体验** → [客户端仪式复刻](client-ceremony.md)：金色仪式感，零成本继承现有 `lol-gold`，登场聚光灯动画
- **实用主义、数据优先** → [极简数据流](minimal-data.md)：去发光去装饰，分析面板升为主角，单琥珀强调统一数据焦点

## 选定后

每份方案第 8 节"迁移成本评估"给出了分阶段实施步骤（如玻璃深空的 P1-P4、赛博朋克的 8 步逐组件替换）。选定一份后即可按步骤实施；也可混合（如"极简数据流布局 + 玻璃深空质感"）。
