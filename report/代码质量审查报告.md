## P0 级别问题 (Critical - 必须修复)

### 1. 安全性：IPC 输入验证不足

**位置**: `src/main/ipc/dataHandler.js:48`

**问题描述**:
```javascript
const raw = fsDep.readFileSync(filePaths[0], 'utf-8')
const data = JSON.parse(raw)
```

直接解析用户提供的 JSON 文件，没有：
- 文件大小限制（可能导致内存溢出）
- 格式验证（可能导致注入恶意数据）
- 内容结构检查

**修复建议**:
```javascript
// 添加文件大小限制
const MAX_FILE_SIZE = 5 * 1024 * 1024 // 5MB
const stats = fsDep.statSync(filePaths[0])
if (stats.size > MAX_FILE_SIZE) {
  return { success: false, error: 'File too large' }
}

// 添加 schema 验证
const data = JSON.parse(raw)
if (!isValidBPSnapshot(data)) {
  return { success: false, error: 'Invalid data format' }
}
```

---

### 2. 类型安全：过多类型断言绕过检查

**位置**: 多处，包括 `src/App.tsx:88`, `src/contexts/HeroContext.tsx:103`

**问题描述**:
```typescript
const ok = loadSnapshot(result.data as BPSnapshot)
const heroes = result.data as Hero[]
```

类型断言绕过了 TypeScript 的类型检查，可能在运行时出错。缺少运行时类型验证。

**修复建议**:
```typescript
// 添加运行时类型检查
function isValidBPSnapshot(data: unknown): data is BPSnapshot {
  if (typeof data !== 'object' || !data) return false
  const snap = data as Record<string, unknown>
  return typeof snap.currentPhase === 'number' &&
         typeof snap.isComplete === 'boolean' &&
         isValidTeamState(snap.blueTeam) &&
         isValidTeamState(snap.redTeam)
}

const ok = loadSnapshot(result.data)
```
